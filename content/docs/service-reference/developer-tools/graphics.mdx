---
title: Graphics
description: Stream graphics and GPU sampling data.
---

Graphics is a DVT client built on `RemoteServerClient`. It opens the `com.apple.instruments.server.services.graphics.opengl` channel.

## Use It For

- Connect through the DVT remote server.
- Call the service-specific DTX methods.
- Use the matching `idevice-tools` command when one exists.

## Feature Flag

```toml title="Cargo.toml"
[dependencies]
idevice = { version = "0.1.64", features = ["usbmuxd", "graphics"] }
```

## Service Names

| Path | Value |
| --- | --- |
| DVT Lockdown | `com.apple.instruments.remoteserver` |
| DVT Lockdown fallback | `com.apple.instruments.remoteserver.DVTSecureSocketProxy` |
| DVT RSD | `com.apple.instruments.dtservicehub` |
| DVT channel | `com.apple.instruments.server.services.graphics.opengl` |

## Rust Example

```rust title="src/main.rs"
use idevice::dvt::{
    RemoteServerClient,
    graphics::GraphicsClient,
};

let mut remote = RemoteServerClient::connect(&provider).await?;
let mut client = GraphicsClient::new(&mut remote).await?;
client.start_sampling(1.0).await?;
let sample = client.sample().await?;
println!("{sample:#?}");
```

## CLI Equivalent

```bash title="Terminal"
idevice-tools graphics
```

## Common Errors

| Error | What to check |
| --- | --- |
| Channel creation failed | The DVT remote server did not accept the channel identifier. |
| Protocol decode error | Check that the client and device service are using the expected DTX payload shape. |
| Developer service unavailable | Start from the iOS 17+ tunnel path or a device state that exposes developer services. |
