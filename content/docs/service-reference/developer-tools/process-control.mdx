---
title: Process Control
description: Launch, kill, and change process memory limit behavior.
---

Process Control is a DVT client built on `RemoteServerClient`. It opens the `com.apple.instruments.server.services.processcontrol` channel.

## Use It For

- Connect through the DVT remote server.
- Call the service-specific DTX methods.
- Use the matching `idevice-tools` command when one exists.

## Feature Flag

```toml title="Cargo.toml"
[dependencies]
idevice = { version = "0.1.64", features = ["usbmuxd", "dvt"] }
```

## Service Names

| Path | Value |
| --- | --- |
| DVT Lockdown | `com.apple.instruments.remoteserver` |
| DVT Lockdown fallback | `com.apple.instruments.remoteserver.DVTSecureSocketProxy` |
| DVT RSD | `com.apple.instruments.dtservicehub` |
| DVT channel | `com.apple.instruments.server.services.processcontrol` |

## Rust Example

```rust title="src/main.rs"
use idevice::dvt::{
    RemoteServerClient,
    process_control::ProcessControlClient,
};

let mut remote = RemoteServerClient::connect(&provider).await?;
let mut client = ProcessControlClient::new(&mut remote).await?;
let pid = client.launch_app("com.example.app", None, None, false, true).await?;
client.disable_memory_limit(pid).await?;
```

## CLI Equivalent

```bash title="Terminal"
idevice-tools process_control launch com.example.app
```

## Common Errors

| Error | What to check |
| --- | --- |
| Channel creation failed | The DVT remote server did not accept the channel identifier. |
| Protocol decode error | Check that the client and device service are using the expected DTX payload shape. |
| Developer service unavailable | Start from the iOS 17+ tunnel path or a device state that exposes developer services. |
