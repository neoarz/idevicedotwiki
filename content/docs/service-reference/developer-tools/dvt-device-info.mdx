---
title: DVT Device Info
description: Read DVT device info, running processes, hardware, and network details.
---

DVT Device Info is a DVT client built on `RemoteServerClient`. It opens the `com.apple.instruments.server.services.deviceinfo` channel.

## Use It For

- Connect through the DVT remote server.
- Call the service-specific DTX methods.
- Use the matching `idevice-tools` command when one exists.

## Feature Flag

```toml title="Cargo.toml"
[dependencies]
idevice = { version = "0.1.64", features = ["usbmuxd", "device_info"] }
```

## Service Names

| Path | Value |
| --- | --- |
| DVT Lockdown | `com.apple.instruments.remoteserver` |
| DVT Lockdown fallback | `com.apple.instruments.remoteserver.DVTSecureSocketProxy` |
| DVT RSD | `com.apple.instruments.dtservicehub` |
| DVT channel | `com.apple.instruments.server.services.deviceinfo` |

## Rust Example

```rust title="src/main.rs"
use idevice::dvt::{
    RemoteServerClient,
    device_info::DeviceInfoClient,
};

let mut remote = RemoteServerClient::connect(&provider).await?;
let mut client = DeviceInfoClient::new(&mut remote).await?;
let processes = client.running_processes().await?;
println!("{processes:#?}");
```

## CLI Equivalent

```bash title="Terminal"
idevice-tools device_info processes
idevice-tools device_info hardware
```

## Common Errors

| Error | What to check |
| --- | --- |
| Channel creation failed | The DVT remote server did not accept the channel identifier. |
| Protocol decode error | Check that the client and device service are using the expected DTX payload shape. |
| Developer service unavailable | Start from the iOS 17+ tunnel path or a device state that exposes developer services. |
