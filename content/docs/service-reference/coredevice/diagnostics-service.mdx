---
title: Diagnostics Service
description: Capture sysdiagnose through CoreDevice Diagnostics Service.
---

Diagnostics Service is a CoreDevice service reached through RSD and RemoteXPC. The client connects to `com.apple.coredevice.diagnosticsservice`.

## Use It For

- Connect through the iOS 17+ CoreDevice tunnel.
- Call the typed CoreDevice client methods.
- Use the matching tool command when one is registered.

## Feature Flag

```toml title="Cargo.toml"
[dependencies]
idevice = { version = "0.1.64", features = ["usbmuxd", "rsd", "core_device"] }
```

## Service Names

| Path | Value |
| --- | --- |
| RSD | `com.apple.coredevice.diagnosticsservice` |

## Rust Example

```rust title="src/main.rs"
use idevice::{RsdService, core_device::DiagnostisServiceClient};

let mut client: DiagnostisServiceClient<_> = rsd_handshake
    .connect(&mut rsd_provider)
    .await?;
let response = client.capture_sysdiagnose(false).await?;
println!("{}", response.preferred_filename);
```

## CLI Equivalent

```bash title="Terminal"
idevice-tools diagnosticsservice
```

## Common Errors

| Error | What to check |
| --- | --- |
| RSD service not found | The connected device did not advertise the CoreDevice service name. |
| RemoteXPC error | Check the CoreDevice action and payload shape used by the client. |
| Tunnel unavailable | Create the iOS 17+ tunnel before connecting to CoreDevice services over USB. |
